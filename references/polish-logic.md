# 润色判断逻辑

> 不是规则，是遇到具体问题时用的"判断脚手架"。遇到类似情况，走一遍看看。

## 情绪判断

### 这段有没有情绪？

读一遍。合上眼。脑子里剩下的是什么感觉？

- **有明确情绪**（愤怒/悲伤/紧张/喜悦）→ 找到是哪句话在传递这个情绪，**那句话是这段的灵魂，改其他句也别动它**
- **没有明确情绪**（就是在推进剧情/描述场景/角色移动）→ 不追求情绪，只保证语言通顺
- **情绪混乱**（前两句愤怒，第三句突然无关描写）→ 可能是原文写乱了。保留情绪主线，把跳出情绪线的句子精简/归位

### 这句改完情绪变了吗？

默读原文→默读改后→对比感觉：

| 原文感觉 | 改后感觉 | 结论 |
|---------|---------|------|
| 愤怒 | 愤怒（但更流畅了） | ✅ 通过 |
| 愤怒 | 有点伤感 | ❌ 情绪跑了，撤回 |
| 愤怒 | 愤怒（但更有力了） | ⚠️ 警惕——如果"更有力"是因为加戏，撤回 |
| 没有感觉 | 没有感觉（但读着顺了） | ✅ 过渡段落，这已经是好的改法 |

## 人感判断

### 这句像人说的吗？

```
默读这句话，想象朋友在咖啡馆跟你说同样的事。
他会不会这么说话？
```

- **会** → 人感OK
- **不会，他可能会说得更碎/更短/有卡顿** → 试着往更碎/更短改
- **不会，他根本不会这么表达** → 说明原文就是书面语，看你判断这个角色/场景是否该说人话
  - 日常对话→必须像人说的
  - 旁白/描述→可以书面一点，但不能太做作
  - 历史文/仙侠文→可以有古风，但不能"看起来是在背诵"

### 改完后还有"人味儿"吗？

**人味儿 = 有点小毛病的自然语言**

原文有毛边的地方，如果：
- **毛边不影响理解** → 留着（"他沉默了一下，然后缓缓开口说" 比 "他开口" 有人味儿）
- **毛边影响阅读**（真的读不通）→ 修掉
- **毛边是角色的语言习惯** → 不但不能修，还要保护（角色就是爱说"然后……然后……"，改了就不像他）

## 直接描写判断

### 这句话是在"给情绪配字幕"吗？

常见的配字幕写法：握紧拳头、咬紧牙关、嘴角上扬、眉头紧锁、眼神坚定、脸色发白、身体发抖、声音颤抖……

这些写法本身不是问题，问题是 **AI润色者看到情绪句就习惯性配上这类描写**，一段情绪描写配2-3个直接动作，读起来像在给角色的每个表情打标签。

```
遇到一句情绪句 → 打算改的时候：
  第一反应是"他握紧拳头""他咬紧牙关"？
  → 停。这是最AI的改法。
  → 换思路：不用直接描写，用别的法子。

别的方法：
  ① 环境烘托：
     - "他很紧张" → "会议室里安静得只剩空调的嗡嗡声"
     - "她很害怕" → "走廊的灯忽明忽暗，她盯着那盏灯，不敢移开视线"
  ② 他人侧写：
     - "他很难过" → "老李从没见过这小子这个样子，连烟都忘了抽"
  ③ 对话弦音：
     - "他很生气" → "你再说一遍。"
  ④ 行动结果：
     - "他生气了" → "他推开门，门撞在墙上，墙灰簌簌往下掉"

如果原文本身就有2-3处直接描写？
→ 选最重要的1处留下，其余的删或换成上面四种之一。
→ 不能确定哪处最重要？留第一个。

如果是自己写润色时：
→ 每300字最多用1处直接神态描写，超过就超标。
→ 一句话里不要叠描写："他握紧拳头，眼神冰冷，声音发颤" → 选一句，其余砍掉。
```

### 快速自检

改完后，全文扫描一遍直接的神态/肢体描写：

```
0-1处 → 正常
2处 → 有点多了，看有没有同质化的（两个"握拳"？两个"咬牙"？）
3处以上 → 必须砍到2处以下。优先留不常见的那个，删掉套路化的。
```

## 句式判断

### 这句太啰唆吗？

删掉一个词，意思变了吗？

```
没变 → 删
变了但影响不大 → 可以删，也可以留着（如果原文需要这种啰唆的节奏感）
变了且伤了原意 → 不删
```

### 这句话读着不顺吗？

```
读一遍（出声默读都行）
↓
读的时候有没有哪个词/哪个分句让你"磕巴"了一下？
↓
没有 → 句子没问题
有 → 找到那个磕巴点，只改那一处
    改完再读，不磕巴了 → 就可以了
     还磕巴 → 可能是句式本身的问题，整个重构
```

**常见磕巴原因**：
- 定语太长（"那个穿着一件深灰色大衣看起来大概四十多岁的中年男人" → 拆开）
- 否定嵌套（"他并不是没有考虑到这个可能性" → "他考虑过这个可能"）
- 多项并列（用"XX、XX和XX"没问题，用"XX、XX、XX以及XX的XX"就不行）

## 判断"放不放过"

遇到一处觉得"好像可以改，但不确定"的句子：

```
1. 不动它，读者会注意到吗？
   → 不会 → 放过
   → 会 → 看2

2. 改完，读者会注意到我改了吗？
   → 不会 → 可以改（改得像没改过一样）
   → 会 → 看3

3. 改完好还是不改好？
   → 改完好 → 改，但要确保"好但不显"
   → 不改好 → 放过
   → 不确定 → 放过（不确定=不成熟=别动）
```

**原则**：每次犹豫，优先选择放过。放过不扣分，改坏了扣分。
