# 网文文笔润色 (Novel Polish)

## 核心：两问

每一处修改，都要先过这两关：

1. **情感保真？** ——原文的情绪还在吗？有没有变淡、变味、变调？
2. **人感自然？** ——改完读起来像人在认真说话吗？还是像机器在优化文本？

两关都过，才改。一关不过，就撤回。

## 工作流

### 第一步：通读，找情绪核

快速读一遍，不看文字细节，只找这段的**情绪核心**。

| 情绪核 | 判断方法 | 例子 |
|--------|---------|------|
| 愤怒 | 节奏是冲的还是压的？句子长还是短？ | 愤怒一般不废话，直接质问或动作 |
| 悲伤 | 是爆发的还是压抑的？有留白吗？ | 悲伤时话少，动作慢，会有停顿 |
| 紧张 | 用词紧凑还是松散？信息密度高不高？ | 紧张时短句多，修饰少 |
| 平淡过渡 | 就是单纯在推进剧情？ | 这种段落不追求情绪，保证流畅就行 |

> **如果读三遍还找不到情绪核 → 说明这段本身就是功能性叙述，只改病句，不动情感。**

### 第二步：润色测评（给用户看）

通读后，做一次快速问题标注，展示给用户。不是V1那种6维表格，而是简洁的**问题点扫描**：

```
## 润色测评

情绪核：[愤怒/悲伤/紧张/平淡]
建议深度：[微调 / 顺稿 / 润色]

问题标注：
- 第2-3行：同义重复（"非常生气"+"无比的愤怒"）
- 第4行：主语重复（连续4个"他"开头）
- 第5行：连续直接神态堆砌（握拳+发抖+嘴唇动，3处→保留1处）
- 第8行：对话太书面（"你难道不知道……"）

建议：顺稿级修改，约10-12处
是否开始？→ [是] [调整深度] [跳过直接改]
```

**测评要点**：
- 情绪核一句话说明 → 你和用户共识了"这段在说什么"
- 问题标注只写**具体位置+具体问题**，不用维度分类
- 问题数控制在**3-5条**，挑最值得说的，别列所有
- 建议深度参考第二步的判断逻辑
- **必须等用户确认**：用户说"可以"才动手，说"轻一点/重一点"就调深度

> 用户也可说"直接改"跳过测评，一步到位。

### 第三步：判断干预深度

根据情绪核和原文质量决定改多少：

| 干预级别 | 情感处理 | 语言处理 | 修改量 |
|---------|---------|---------|--------|
| **微调** | 不动 | 只改真的读不通的（病句、搭配错误） | ≤5% |
| **顺稿** | 保留，只去掉妨碍阅读的冗余 | 删同义重复、精简啰唆的句式 | 5%-15% |
| **润色** | 情绪表达不到位可补1-2处，不能改变情绪基调 | 句式优化、精准用词、调整节奏 | 15%-30% |

> 干预深度取决于原文质量，不取决于你"想改多少"。

### 第四步：逐句动手（两问循环）

每一句，按这个流程走：

```
看这句
↓
默读一遍，感受：这句的情绪是什么？
↓
有没有问题？（读不通？太啰唆？情绪不对？读了出戏？）
↓
没有 → 跳过（不改就是最好的改法）
有 → 想一种改法
    ↓
    改完默读：情绪还在吗？
    ↓
    在 → 再默读：像人说的吗？
        像 → 确认修改
        不像 → 换一种改法
    不在 → 撤回，换更轻的改法
```

**改句原则**：
- **不要表现你"改得好"**，要表现"这话就是这个作者写的，只是更顺了"
- **改不动就放弃**：十处里有一两处犹豫不决的，放过它们
- **不要换词显水平**：原文"非常生气"不一定要换成"怒不可遏"，如果"非常生气"符合作者的说话习惯，它就是对的

### 第五步：全文朗读验证 + 输出

改完后，从头到尾默读一遍，问三个问题：

```
1. 这还是同一个人在说话吗？ （风格一致）
2. 读起来别扭的地方有几处？  （目标：0处）
3. 如果我是读者，我会觉得"改得真不错"还是"感觉怪怪的"？
   （前者→可以。后者→哪里怪？回退重改那一处。）
4. （专项检查）全段直接神态描写有几处？
   ≤1处 → ✓
   2处 → 考虑删一处
   ≥3处 → 保留最关键的一处，其余换成烘托/侧写/删掉
```

只要问题3的答案是"怪怪的"，不管前两个多好，都说明改太过了。

## 改什么 / 不改什么

### 可以改（按优先级）

1. **真的读不通的**：成分残缺、搭配不当、标点错误
2. **同义重复**："慢慢地缓缓地"、"感到愤怒和生气"
3. **读到出戏的**：突然冒出来的书面语（角色说书面语不算）、突然跳出来的成语
4. **情绪表达明显不到位**：高潮场景的"他很开心"→可以用动作展示，但只改这一处
5. **过度直接的神态描写堆砌**：见下方说明

### 不能改（底线）

| 不能动 | 为什么 |
|--------|--------|
| 带情绪的短句 | "走。" "滚。" "别说了。" ——这种句子的力量在于短，别加长 |
| 角色的个性对话 | 哪怕是病句——如果这个角色说话就是结巴/粗鲁/啰唆，那是他的风格 |
| 作者的情绪节奏 | 原文该抒情的时候写了一大段，该快的时候一笔带过——那是作者的设计 |
| 功能性过渡 | 角色从A走到B的路上，写什么不重要，通顺就行 |
| 有瑕疵但有人感的句子 | "他沉默了一会儿，然后说——"不如"他说"精炼，但像真实的人类叙述 |

### 关于神态描写的专项：不要直接写，善用烘托和侧写

这是当前AI润色最容易翻车的地方。看到一句"他很生气"，AI往往会改成"他握紧拳头，额角青筋暴起，牙齿咬得咯咯作响"——这就是典型的表情配字幕。

**区分三种写法**：

| 写法 | 例子 | 评价 |
|------|------|------|
| ❌ 直接的神态堆砌 | "他握紧拳头，眉头紧锁，眼神冰冷" | AI味最重，每个表情都在给读者做注释 |
| ⚠️ 克制的直接描写 | "他握紧了拳头"（只一处，且有用） | 可以用，但全段不超过1处 |
| ✅ 烘托 | "房间里忽然安静了。连窗外的蝉都不叫了。" | 好，用环境传递情绪 |
| ✅ 侧写 | "老张在江湖上混了三十年，从来没见过一个人的脸色能变成这样。" | 好，通过他人的视角传递震撼 |

**判断原则**：

```
遇到一段情绪描写 → 数一数直接写了几个神态/动作：
0-1处 → 正常，不处理
2处 → 考虑留一个删一个
3处以上 → 执行手术：留最关键的一处，其余换成烘托或侧写

如果自己动手改时：
- 第一反应是"他握紧拳头""他咬紧牙关" → 停。这不是好改法
- 问自己：不用这些直接描写，能不能让读者感受到情绪？
  - 用环境 → "酒杯在桌上颤了一下"
  - 用他人 → "小师妹从没见过师兄这个样子"
  - 用对话弦音 → "你再说一遍。"
  - 用行动结果 → "他推开门，门撞在墙上，墙灰簌簌地往下掉"
```

## 输出

### 核心输出：润色后文本

```
改好了。改了X处。
（原文情绪：[悲伤/愤怒/紧张/平淡] → 保留 ✓）
（自测：读起来像同一个人在说话 ✓）
（神态描写：≤1处 ✓）

[润色后的全文]
```

### 润色后自查清单

输出前，快速过一遍：

```
□ 情绪变味了吗？原文的愤怒改完了变成悲伤？撤回
□ 有不应该改的句子被改了吗？
  带情绪的短句？角色的个性对话？作者的情绪节奏？
□ 全段直接神态描写≤1处？超标了砍到1处
□ 读起来像同一个人在说话？
  如果有1秒的迟疑→那一处改太过了，回退
□ 修改率超标了吗？
  微调≤5% | 顺稿≤15% | 润色≤30%
```

> 全部勾选 ✓ → 输出 | 任何一项不通过 → 回退对应修改，重改

### 可选总结（用户要求时）

挑最值得说的 **1-2处**（不是所有修改），每处只写三行：

```
1. 第X段：「原文」→「改后」
   为什么改：这句话读不通 / 情绪没到位 / 太啰唆
   怎么想的：原文想表达……但写的时候绕了一下，顺过来就好

2. 第X段：「原文」→「改后」
   为什么改：这句书面味太重
   怎么想的：这个角色不会这么说话，改成……就自然了
```

## 多轮交互

用户不满意 → 走以下路径：

| 路径 | 用户说 | 处理方式 |
|------|--------|---------|
| **局部重改** | "第三段改过了，恢复原样重来" | 仅回退指定段落 → 降低一档强度 → 重改 |
| **降档/升档** | "深度太过了，用标准重来" | 保持对原文的理解不变 → 切换强度 → 重改 |
| **版本回退** | "上一版比现在好" | 回到修改前的版本（保留原文快照） |
| **指定方向** | "只修病句，其他不改" | 仅做对应维度的修改，其余跳过 |
| **不满意但说不清** | "感觉不对，但说不出来" | 逐条读自查清单让用户指认"哪条感觉不对" → 定位问题后走前面路径 |

> **默认策略**：每次输出前保留原文快照。用户要求"再来一版"时始终从原文重改，不要对已润色文本二次润色（防止层层叠加变味）。

## 边界情况

- **过短（<200字）**：样本太小，只做微调
- **过长（>5000字）**：按段落分批，每批1000-1500字
- **AI生成文本**：D档，但也得先找情绪核。AI文本往往没有情绪，别强加情绪进去
- **用户提供风格提示**：以用户说的为准。他说"这是历史文"就别按都市文改
- **文本质量已经很好**：只改0-2处显而易见的硬伤，其余不动

## 风格预设（可选）

用户可以预置自己的写作风格，后续使用时自动载入：

```
【风格预设卡】
题材偏好：[都市/玄幻/历史/悬疑/通用]
节奏偏好：[快/中/慢/自动识别]
默认干预级别：[微调/顺稿/润色]
风格标签：[如"写都市爽文的，对话要快要燃"]
```

使用方式：
- 用户说"帮我记住，我写都市爽文，节奏快，重点改语病" → 建立预设卡
- 建立后后续每次使用自动载入，与文本识别结果合并判断
- 没有预设时完全依赖文本自动识别

## 与 novel-audit 联动

`novel-audit`（网文审计）发现文笔层面的问题后，可直接传入本技能做专项处理：

| 审计发现 | 传入本技能后 |
|---------|-------------|
| AI痕迹重（套路句式多） | D5去AI痕迹 → 口语化改造 + 打破对称结构 |
| 对话同质化（去掉标签分不出谁在说话） | 对话专项处理 → 差异化 + 口语化 |
| 节奏问题（拖沓/仓促） | 节奏调整 → 压缩/展开 |
| 情节/人物问题 | 不处理——本技能不做情节和人物层面的改动 |

> 联动数据仅在当次会话中传递，不存储。用户说"刚刚审计的那段帮我润色" → 参考审计报告中的文笔问题分布，按对应维度执行。
